DETAILED ROUTING DECISION TABLE
================================

Scenario 1: MVP Chatbot (Week 1)
────────────────────────────────
├─ Requirement: Simple chat + read files/git
├─ Protocol: MCP
├─ Setup:
│  ├─ Install @modelcontextprotocol/sdk
│  ├─ Choose MCP server (filesystem, git, fetch)
│  ├─ Create /api/chat route
│  └─ Connect to Vercel AI SDK useChat hook
├─ Time: 2-3 days
├─ Complexity: Low ⭐
└─ Scaling: Use later + A2A wrapper


Scenario 2: Team Workspace (2-3 weeks)
──────────────────────────────────────
├─ Requirement: Multi-agent delegation, task routing
├─ Protocol: A2A
├─ Setup:
│  ├─ Get A2A credentials/registry access
│  ├─ Install A2A SDK (Node.js)
│  ├─ Define Agent Cards (JSON schema)
│  ├─ Create /api/agents endpoint
│  ├─ Implement agent discovery
│  └─ Add task delegation handler
├─ Time: 7-10 days
├─ Complexity: High ⭐⭐⭐
└─ Scaling: Native support for teams


Scenario 3: Multi-Channel Distribution (Weeks 2-4)
──────────────────────────────────────────────────
├─ Requirement: Chat app + Discord/Slack/Zalo
├─ Protocol: GoClaw
├─ Setup:
│  ├─ Deploy GoClaw (Docker or binary)
│  ├─ Set up PostgreSQL
│  ├─ Register agents in GoClaw
│  ├─ Create /api/ws WebSocket route
│  ├─ Implement custom bridge client
│  └─ Connect Playground UI to WS
├─ Time: 5-7 days (+ infra)
├─ Complexity: Medium ⭐⭐⭐
└─ Scaling: 5 channels out-of-box


Scenario 4: Privacy-First Decentralized (Weeks 2-4)
───────────────────────────────────────────────────
├─ Requirement: Self-hosted, federated, no SaaS
├─ Protocol: Matrix
├─ Setup:
│  ├─ Deploy Matrix homeserver (Synapse/Conduit)
│  ├─ Register bot user
│  ├─ Use matrix-bot-sdk (TypeScript)
│  ├─ Create playground room
│  └─ Connect frontend via Matrix SDK
├─ Time: 4-6 days (+ homeserver ops)
├─ Complexity: Medium ⭐⭐⭐
└─ Scaling: Federated/decentralized


Scenario 5: Everything (Advanced, Weeks 4+)
───────────────────────────────────────────
├─ Requirement: Tools + teams + channels + decentralized
├─ Protocol: MCP + A2A + Matrix
├─ Stack:
│  ├─ MCP for single-agent tools
│  ├─ A2A for multi-agent delegation
│  ├─ Matrix for decentralized chat
│  └─ GoClaw as optional gateway
├─ Time: 10-14 days
├─ Complexity: Very High ⭐⭐⭐⭐⭐
└─ Scaling: Best of all worlds

RISK ASSESSMENT
===============

MCP Risks:
├─ Single-agent only → add A2A later
├─ Local spawn overhead → wrap in REST gateway
└─ Low risk overall (proven, widespread)

A2A Risks:
├─ Overkill for MVP → start with MCP
├─ Vendor registry required → not fully decentralized
└─ Moderate risk (promising but newer)

GoClaw Risks:
├─ Message size limit (512KB) → chunk large responses
├─ Go binary only → no Node.js official support
└─ Medium risk (proven but smaller ecosystem)

Matrix Risks:
├─ Homeserver ops overhead → use managed service
├─ Steeper learning curve → more documentation needed
└─ Medium risk (stable but infrastructure-heavy)

OpenClaw Risks:
├─ Node.js support unofficial → Python/Go primary
├─ Smaller ecosystem → fewer integrations
└─ Medium risk (maturing, community-driven)
