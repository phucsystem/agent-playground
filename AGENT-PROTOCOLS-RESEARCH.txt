═══════════════════════════════════════════════════════════════════════════════
  KNOW YOUR CONSTRAINTS
═══════════════════════════════════════════════════════════════════════════════

MCP Constraints:
├─ Process spawn overhead: 50-200ms per request
├─ Solution: Reuse connections or REST gateway
└─ Cost: Time, not money

A2A Constraints:
├─ Agent discovery latency: 200-500ms
├─ Solution: Cache agent list, refresh every 5min
└─ Cost: Latency adds up for real-time

GoClaw Constraints:
├─ Message size: 512KB limit
├─ Solution: Chunk large responses
└─ Cost: Multiple API calls for large content

Matrix Constraints:
├─ Homeserver ops: 2-3 weeks setup for scale
├─ Solution: Use managed service or lightweight Conduit
└─ Cost: Infrastructure complexity

═══════════════════════════════════════════════════════════════════════════════
  RISKS TO AVOID
═══════════════════════════════════════════════════════════════════════════════

❌ Build hybrid stack (MCP + A2A + Matrix) from day 1
   → Takes 10-14 days, debug nightmare
   → Recommendation: Start MCP, add A2A week 2, scale later

❌ Use XMPP for new projects
   → Agent support deprecated, XML verbose, small community
   → Recommendation: Use Matrix instead (modern, decentralized)

❌ Spawn new MCP process per request
   → 50-200ms overhead per spawn
   → Recommendation: Reuse connections or REST gateway

❌ Don't plan for multi-tenancy
   → User data leaks between sessions
   → Recommendation: Use isolated channels/agents per tenant

❌ Deploy Matrix without scaling plan
   → Default Synapse = 2GB RAM, breaks at 10 users
   → Recommendation: Use Conduit (Rust, 40MB) or managed service
